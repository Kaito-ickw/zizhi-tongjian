# 画像生成スタイルガイドライン (instruction-gen-image.md)

本ドキュメントは、『資治通鑑 現代日本語訳プロジェクト』において、AI画像生成（Nano Banana 2 / Imagen 3系列）を行う際のスタイル統一とプロンプト指定に関するガイドラインです。

---

## 1. 共通の制約ルール

### ① 画像内言語の厳格な制御
*   **地図および文物図解**: 文字やラベルを含む必要がある場合は、**必ず日本語**で表記します。簡体字や大陸風の書体を避け、厳密に日本の漢字標準（JIS）や明朝体・ゴシック体・教科書体を使用するようにプロンプトで制御してください。
*   **挿絵およびポートレート**: 原則として**文字情報を一切含まない（No text or letters）**ビジュアル単体で構成します。中国語の毛筆サインや題名が自動生成されないよう、「no text, no signatures, no calligraphy」などのネガティブプロンプト指定や指示を徹底してください。

---

## 2. 種類別の画像スタイル定義

### 【カテゴリーA：地図・勢力図 (Maps)】
歴史的な地形や国家・クランの勢力割拠を示す地図。
*   **質感・背景**: 暖かみのあるセピア色の「アンティーク羊皮紙（Parchment）」の質感。
*   **枠線**: なし（フレーム装飾などは設けず、紙の端まで自然に描く）。
*   **シンボル**: 非常にシンプルにする。都市位置に宮殿や城のミニチュアアイコンは配置せず、単なる四角形マークやラベル線のみで表現する。
*   **方位磁針**: 細い線で描かれたシンプルで控えめなデザイン。
*   **文字の筆致**: かすれがあり、力強い手書きの「和風筆文字・行書風（Japanese Calligraphy style）」。
*   **彩色**: 原則として単色トーンベースだが、勢力範囲を識別しやすくするため「薄い透過的な地域色分け（Subtle color-coding for territories）」は許容する。

---

### 【カテゴリーB：歴史的名場面の挿絵 (Scenes)】
劇的な事件や戦況を描く挿絵。
*   **描画スタイル**: 伝統的な水墨画調（Traditional Ink and wash painting）。
*   **構図**: ダイナミックな鳥瞰・俯瞰構図（例：川と城壁、遠景の山、手前の人物などの位置関係を立体的に捉える）をベースとします。
*   **描き込み度**: **中程度（Medium-light detail）**。最初のサンプル（Pattern A）の構図や配置を踏襲しつつ、過密なレンガ・甲冑のテクスチャや密な雨線などを排除します。太めの筆跡や柔らかい墨のぼかしを多用し、シンプルかつ芸術的に要素を表現します。

---

### 【カテゴリーC：人物ポートレート (Portraits)】
キーパーソンのビジュアル。
*   **描画スタイル**: 伝統的な「写意人物画（Traditional minimalist ink painting style）」に近い水墨ポートレート。
*   **描き込み度**: **あっさりとした描写（Minimalist details）**。骨格やしわをリアルに描き込む劇画調（Pattern A）は避け、最小限のシンプルで流麗な墨線と薄墨のぼかしのみで人物の佇まいと意志を表現します。
*   **背景**: 余白（すっきりとした無地の背景）を大きく取ります。

### 【カテゴリーD：文化・文物・装備の図解 (Artifacts)】
儀礼具（繁纓など）や城壁・堤防の構造などのビジュアル解説。
*   **レイアウト**: 博物館の目録や図鑑（Museum catalog / Diagram）のような、白〜薄い背景上のすっきりとした配置。
*   **描画スタイル**: 細い線と繊細な水墨の陰影を用いたスケッチ調。
*   **テキスト配置**: 図の上部または下部に、明瞭な日本語ラベルと解説文を添える。

## 3. 画像生成の運用・管理プロセス

### 3.1 運用契約 (I/O)
*   **入力**: 対象年の `translation_full` および胡注。
*   **出力**: `docs/images/卷NNN/jNNN_yMM_slug.jpg`。
*   **登録**: `data/kb/卷NNN/jNNN_yMM.json` の `illustrations` 配列に登録。**※`translation_full` 本文は一切編集しないこと。**

### 3.2 ファイル制約・枚数バジェット
*   **画像解像度**: 長辺 ≤ 1600px。
*   **ファイルサイズ**: JPEG形式、300KB 以下目安。
*   **枚数制限**: 1年あたり最大4枚程度。
*   **圧縮と最適化**: リポジトリ肥大防止のため、画像保存後は必ず `pipeline/image_sync.py` を使用して圧縮を行うこと。

### 3.3 画像内日本語テキストの検証
*   生成モデルは日本語の漢字を崩しやすいため、生成後にテキストの可読性を必ず検証する。
*   文字が崩れている場合は再生成を試みる。
*   どうしても改善しない場合は、画像内テキスト無しの構成（No text）にし、キャプション（caption）側で説明を記述する運用にフォールバックすること。

### 3.4 モデル/生成手順の固定
*   **使用モデル名**: Gemini (Imagen 3 / Nano Banana 2 等) を明示する。
*   **生成コマンド**: 再現可能性を担保するため、`agy` 等の実行コマンド（プロンプトや各種オプション設定を含む）を明記・記録する。

### 3.5 バリアント選定
*   同一プロンプトや異なるパラメータで複数枚の画像を生成し、最も品質が良くスタイルに合致する1枚を確定する。
*   確定版のみをリポジトリ（Git）に追加し、選定から外れた不要なバリアント画像は残さない。

### 3.6 来歴・ライセンス
*   AI生成画像である旨と使用したモデル名を来歴として明記・記録する。
*   成果物のライセンスは CC BY-NC-SA 4.0（表示 - 非営利 - 継承）とする。

### 3.7 illustrations[] スキーマ定義 (SHARED SCHEMA)
kb record 内の `illustrations` フィールドのスキーマ定義：
```json
kb record OPTIONAL top-level "illustrations": [ { "slug":"map", "category":"A" (A=map/B=scene/C=portrait/D=artifact), "file":"jNNN_yMM_slug.jpg" (basename; lives in docs/images/卷NNN/), "caption":"…", "anchor":"translation_full 内の一意な部分文字列。画像はこの直後に挿入される" } ]. Omit the array entirely when there are no images.
```

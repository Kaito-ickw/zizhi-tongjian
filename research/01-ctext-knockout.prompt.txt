あなたは資治通鑑(294巻・約300万字)を現代日本語へ機械翻訳する大規模プロジェクトのデータ調達担当です。
底本ソースとして Chinese Text Project (ctext.org) を第一候補にしています。
着手前の「ノックアウト検証」として、以下を Web 検索と(可能なら)実際のAPI/HTTPアクセスで調査し、判断を出してください。

# 調査項目(ノックアウト判定)
1. ctext.org に資治通鑑の全文(全294巻)が収録されているか。収録範囲・欠落の有無を確認する。
2. ★最重要★ ctext.org に胡三省注(胡三省『資治通鑑音注』)が含まれているか。本文と構造的に対応付け(アライン)されているか、それとも本文のみで注は無いか。注の網羅性も評価する。
3. bulk 取得の経路:
   - ctext には公式 API があるか(ctext.org の tools/api 等)。
   - レート制限、認証(APIキー)、会員制(有料 plus 会員のみの機能)の有無。
   - 利用規約上、全文の bulk ダウンロード/スクレイピングが許容されるか・制限されるか。
   - 実際に API エンドポイントを叩いて資治通鑑のテキストが取れるか確認できれば確認する(可能な範囲で curl 等を使ってよい)。
4. ctext が資治通鑑に用いている底本(どの版・刻本か)。標点・校勘に編集著作権が及ぶ可能性も指摘する。
5. フォールバック源の評価:ctext が胡三省注の点で不十分な場合に備え、以下が「全文 + 胡三省注」を機械可読形式で持つか調べる:
   - 維基文庫 (zh.wikisource.org)
   - 殆知閣 / 中華古籍系コーパス
   - GitHub 上の資治通鑑コーパス(胡三省注付きのもの)
   - その他信頼できる電子テキスト
6. 総合判定:データ調達計画について GO / GO(フォールバック前提) / NO-GO のいずれか。推奨する具体的な調達元とアクセス方法を明記する。

# 出力要件
- 詳細な調査レポートを Markdown でファイル `research/01-ctext-knockout.md` に書き出すこと(日本語)。
- レポート末尾に「## 判定」セクションを設け、GO / GO(フォールバック前提) / NO-GO と、その根拠、推奨調達元・アクセス方法を1段落で明記すること。
- 確認できた事実と、推測・未確認の事項を明確に区別すること。出典 URL を併記すること。
- 最後の応答(stdout の最終メッセージ)には、判定結論と推奨調達元を3〜5行で要約すること。

あなたは資治通鑑(対象時代: 前403年〜後959年 = 戦国・秦・漢・三国・晋・南北朝・隋・唐・五代)を現代日本語訳するプロジェクトの、エンティティ辞書(人物・地名・官職の「表記揺れ → 正規名」マッピング)の初期データ調達担当です。
翻訳の一貫性を担保するため、巻をまたいで同一人物/地名/官職を同定できる「初期辞書」を、信頼できる機械可読ソースから作りたい。着手前の調査として以下を、Web 検索ではなく curl/HTTP で直接一次情報・API・配布リポジトリに当たって調べ、判断を出してください(ネイティブ web 検索は Cloudflare で不可)。

# 調査項目
1. 人物 (persons):
   - CBDB(China Biographical Database, Harvard/AS/PKU)が対象時代をカバーするか。前漢〜唐〜五代の収録密度。
   - CBDB の別名(本名/字/諡/廟号/別号)テーブルの有無と構造。表記揺れ → 正規人物ID の同定に使えるか。
   - 入手形態(SQLite/Access/MySQL ダンプ、API)、更新頻度、ライセンス。
   - フォールバックとして Wikidata(中国史人物の別名・同定)、その他人名典拠。
2. 地名 (place names):
   - CHGIS(China Historical GIS, Harvard)/ TGAZ(Temporal Gazetteer)の対象時代カバー、時代区間付き地名、現代地名へのマッピング。
   - 入手形態・API・ライセンス。
   - その他の歴史地名典拠(中國歷史地名、Wikidata 等)。
3. 官職 (official titles):
   - 歴史的官職名の機械可読典拠は存在するか。CBDB の官職(office)テーブルの利用可否。
   - Hucker "A Dictionary of Official Titles in Imperial China" の機械可読性・権利状況。
   - 王朝ごとに官職体系が変わる問題への対処方針。
4. 各カテゴリについて: 資治通鑑の対象時代での実用カバー率、機械可読性、ライセンス、初期辞書の seed としての具体的な取り込み方法を評価する。
5. 各カテゴリの判定: 「使える初期ソース + 取り込み方法あり」か、「良いソース無し → 本文からの自動抽出でブートストラップするしかない」か。

# 出力要件
- 詳細レポートを Markdown で `research/03-entity-dictionary-sources.md` に書き出す(日本語)。
- 末尾に「## 判定」を設け、人物/地名/官職それぞれの推奨初期ソースとアクセス方法、ライセンス上の注意、初期辞書スキーマ案(エントリ → 正規名 + 異名 + 典拠ID + 時代区間)を明記する。
- 確認済みと未確認を区別し、出典 URL を併記する。
- stdout 最終メッセージに、3 カテゴリの判定結論を 4〜6 行で要約する。
